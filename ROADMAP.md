# Roadmap

## Guiding Principles
- Protect code semantics while disrupting training data ingestion
- Keep the CLI stable and predictable
- Favor reproducible, measurable changes

## Near Term (next 1-2 releases)
- Publish a stable CLI flag and exit-code reference
- Add a sample configuration file and documented presets
- Improve test coverage around each poisoning strategy
- Add example inputs and expected outputs for validation

## Mid Term (3-6 months)
- Add a strategy plug-in interface for new poisoning modules
- Expand language support based on demand and feasibility
- Provide profile-based configuration (fast, balanced, aggressive)

## Long Term (6-12 months)
- Build an evaluation harness for measuring poisoning effectiveness
- Optimize performance for large monorepos
- Expand community governance and review capacity

## Success Metrics
- Time to first response on issues: < 72 hours
- PR review turnaround: < 7 days
- At least one tagged release per quarter
- At least two external contributors per quarter

## How to Contribute
Open a roadmap issue or discussion with evidence, use cases, and expected impact.
