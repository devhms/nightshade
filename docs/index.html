Defense Strategies

Entropy Scrambling

Replaces internal variable names with randomly generated hashes (e.g. var_7f8a2) to destroy semantic context, increasing perplexity for LLMs.

Semantic Inversion

Deterministically maps identifiers to out-of-domain vocabulary (culinary, automotive, biology) to cause severe cognitive dissonance during LLM training.

Control Flow Flattening

Rewrites simple sequential logic into complex while-switch dispatch loops, mutating the code structure to survive normalization passes.

Dead Code Injection

Inserts non-trivial opaque predicates (e.g. if (Math.random() > 10)) and language-specific dead blocks to bloat the AST and misdirect attention.

String Literal Encoding

Bypasses exact-match deduplication pipelines by randomly splitting and encoding string literals using Base64 or Hex patterns.

Cryptographic Watermarking

Embeds a steganographic fingerprint via invisible zero-width spaces, allowing authors to prove provenance if their code appears in an LLM.

Installation & Usage

GitHub Action (CI/CD)

Integrate Nightshade directly into your deployment pipeline to ensure all shipped code is protected automatically.

jobs: protect: runs-on: ubuntu-latest steps: - uses: actions/checkout@v3 - name: Run Nightshade uses: ibrahim-nightshade/nightshade-action@v1 with: input-dir: './src' strategies: 'all' verify: true

CLI Toolkit

Run locally using Java 21 or compile to a native binary using GraalVM.

java -jar nightshade-2.0.0-all.jar --input ./src --strategies flatten,semantic,watermark --verify --report out.md
